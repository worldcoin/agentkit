## Internationalization (i18n)

The UI Kit includes internationalization support through the `I18nProvider` and related utilities.

**Key Exports**:

- `I18nProvider`: Context provider for i18n
- `useI18n`: Hook for accessing translations
- `loadLocale`: SSR-friendly locale loader
- `availableLocales`: List of available locales

**Examples**:

```tsx
// Server-side (Next.js)
import { I18nProvider, loadLocale } from '@worldcoin/mini-apps-ui-kit-react'
// Client-side
import { useI18n } from '@worldcoin/mini-apps-ui-kit-react'

export default async function Page({ searchParams }) {
  const { locale } = await searchParams
  const localeValue = typeof locale === 'string' ? locale.toLowerCase() : 'en'
  const localeData = await loadLocale(localeValue)

  return (
    <I18nProvider localeData={localeData}>
      <AppContent />
    </I18nProvider>
  )
}

function MyComponent() {
  const { t } = useI18n()
  return <div>{t('key')}</div>
}
```

**Edge Cases**:

- Use `loadLocale` for SSR compatibility
- Locale data must be provided to I18nProvider
- Components like PhoneField use i18n for formatting
